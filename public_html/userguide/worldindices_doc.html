World Indices table data scenario


  • Data in world indices table come using JSON API.
  • When BlissWorldIndex.php is call it make call to BlissWorldIndex.js
  • In .js File initially it make call to display_wi_table() function which is use to create table dynamically.
  • From this function we are makeing call to world_indices_connect.php page. where we are taking live data from google and yahoo API.
  • In world_indices_connect.php firstly we are fatching data from yahoo API for various countries live data with there time and date and for some countries data we are using google API Where we are fatching same data.
  • All data comes in file formate which is then extracted to string using "file_get_contents()"this function reads entire file to string.
  • Then json_decode() is use to decode json object to string.
  • this this is string is divided in to multipule value and this values are save in multipule variable.
  • And lastly time is converted into GMT formate and all this data is send back to BlissWorldIndex.js File in form of json object.
  • In .js file this object is decoded and value is displayed in table using display_wi_function().

World Indices table data color coding scenario


  • In world indices page we are using various color to indicate various thing such as:-
  • Grey color :- Indicate market close.
  • White color :- Indicate Live Market.
  • Green color in last 3 column :- Indicate positive market.
  • Red color in last 3 column :- Indicate negative market.
  • To impliment this color various conditions are use.
  • firstly to show live or close market we are comparing last trade time with 15 minutes before and after system time,if last trade time is less then 15 below system time and greater then 15 minutes above system time then market is live else it is close.
  • if market is live then to determine where market is positive or negative we are comparing last trade price with zero,if it is greater then zero positive else negative .
